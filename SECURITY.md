# Security Policy

If you believe you’ve found a security vulnerability, please report it privately as described below.

## Supported versions

We support the **current default branch** (`main`) with security updates. We do not maintain separate release branches or versioned releases.

## Reporting a vulnerability

Please report security issues **privately**. Do not open a public issue.

- **Preferred:** Use GitHub’s flow: **Security** tab → **Report a vulnerability** (or “Advisories” → “Report a vulnerability”). That creates a private draft advisory.
- **Alternative:** Contact the repository owner (see the repo’s About section or maintainer list). If you need another channel, say so in a minimal public issue and we’ll provide one.

We’ll acknowledge your report as soon as we can and will keep you updated. If we accept the report, we’ll work on a fix and coordinate disclosure. If we decline, we’ll explain why. We ask that you allow a reasonable time for a fix before any public disclosure.
